# Audiencias — Guía para Adaptar el Copy y el SEO

## Por qué importa definir la audiencia antes de escribir

El mismo servicio necesita palabras completamente distintas según quién lee. Un gemelo digital para un técnico de mantenimiento industrial y para un director de operaciones son el mismo producto explicado de dos formas que no se parecen en nada.

**La regla:** Escribe para una persona específica, no para "todos los interesados".

---

## Arquetipos de audiencia más comunes en tecnología e IA

### 1. El Decisor de Negocio (CEO, Director, Gerente)

**Lo que le importa:** Retorno de inversión, riesgo, tiempo hasta ver resultados, reputación.
**Lo que no le importa:** Cómo funciona por dentro.
**Lenguaje:** Negocio, eficiencia, coste, ventaja competitiva, escalabilidad.
**Formato preferido:** Resumen ejecutivo, casos de éxito con números, comparativas.

Copy para este perfil:
> "Reduce el tiempo de análisis de datos de 3 horas a 8 minutos. Sin cambiar tu infraestructura actual."

---

### 2. El Técnico o Desarrollador

**Lo que le importa:** Cómo funciona, qué tecnologías usa, integración, documentación, autonomía.
**Lo que no le importa:** Metáforas y storytelling.
**Lenguaje:** API, stack, arquitectura, compatibilidad, código abierto.
**Formato preferido:** Documentación técnica, ejemplos de código, benchmarks.

Copy para este perfil:
> "Arquitectura modular compatible con cualquier entorno Linux. API REST documentada. Sin dependencias propietarias."

---

### 3. El Cliente Final / Usuario No Técnico

**Lo que le importa:** Que funcione, que sea fácil, que resuelva su problema concreto.
**Lo que no le importa:** La tecnología detrás.
**Lenguaje:** Simple, cotidiano, orientado a tareas.
**Formato preferido:** Pasos simples, antes/después, testimonios de personas como él.

Copy para este perfil:
> "Dile a la IA qué necesitas. Ella lo hace. Tú revisas el resultado en 2 minutos."

---

### 4. El Cliente Corporativo o Institucional

**Lo que le importa:** Cumplimiento normativo, seguridad de datos, escalabilidad, soporte, contratos.
**Lo que no le importa:** Precio. Le importa el riesgo y la garantía.
**Lenguaje:** Compliance, SLA, GDPR, auditoría, infraestructura privada, soporte 24/7.
**Formato preferido:** Casos de estudio detallados, white papers, reunión de calificación.

Copy para este perfil:
> "Infraestructura certificada bajo normativa europea. Datos procesados localmente, sin salir de tu jurisdicción. Soporte dedicado con SLA garantizado."

## Cómo adaptar el mismo servicio para dos audiencias

Ejemplo: Servicio de IA para análisis de documentos

**Para el Decisor:**
> Deja de perder horas revisando contratos, informes o expedientes manualmente. Nuestra herramienta los analiza en segundos y te entrega solo lo que necesitas saber.

**Para el Técnico:**
> Pipeline de procesamiento de documentos basado en modelos de lenguaje ajustados por sector. Entrada: PDF, DOCX, imágenes escaneadas. Salida: JSON estructurado o integración directa vía API.

**Para el Usuario Final:**
> Arrastra el documento. La IA lo lee por ti y te dice qué hay dentro. Sin formularios, sin esperas.

## Capas de complejidad técnica — Cuándo mostrar qué

Para proyectos tecnológicos con audiencias mixtas, organiza la información en capas:

**Capa 1 — Para todos (visible siempre):**
Beneficio en lenguaje simple. Qué resuelve y para quién.

**Capa 2 — Para interesados (en la página de servicios):**
Cómo funciona en términos generales. Casos de uso. Proceso simplificado.

**Capa 3 — Para técnicos y corporativos (en documentación o sección dedicada):**
Arquitectura, especificaciones, integraciones, seguridad, normativa.
Aquí puedes usar el lenguaje técnico completo: ACIDE, CPD, LLM, edge computing, GDPR, etc.

**Regla de oro:** La complejidad técnica no desaparece. Se guarda para quien la necesita y la va a valorar. Los demás ven solo el beneficio.
