# Estructura de Contenido — Guía por Tipo de Página

## Principio de la pirámide invertida

En periodismo clásico, la información más importante va primero. En SEO moderno y GEO, esto es ley.

```
[Lo más importante] ← Primera frase / primer párrafo
[Contexto y explicación]
[Detalles adicionales]
[Información de soporte]
[Lo más prescindible]
```

Las IA cortan desde abajo. Los lectores también. Lo que va arriba, siempre llega.

## Estructura: Página Principal (Home)

```
[H1] — Respuesta directa a "¿qué es esto?" + beneficio central
       Máx. 60 caracteres. Contiene la palabra clave principal.

[Subtítulo / tagline] — Una frase de Brand Copy. Emoción o curiosidad.
                        No repite el H1. Amplía o contrasta.

[Párrafo 0 — Hero] — 2-3 frases. Responde: ¿para quién es? ¿qué resuelve?
                     Este texto debe poder usarse como meta description.

[Sección de pilares / servicios]
  H2: [Servicio 1 — orientado a beneficio, no a nombre técnico]
    Párrafo corto (3-4 líneas): qué es, para qué sirve, resultado.
  H2: [Servicio 2]
    ...
  H2: [Servicio 3]
    ...

[Sección de prueba social o credibilidad]
  Casos de uso, logos de clientes, datos propios, citas reales.

[Sección FAQ — mínimo 3 preguntas]
  H2: Preguntas frecuentes sobre [tema principal]
  H3: ¿Pregunta 1?  → Respuesta directa
  H3: ¿Pregunta 2?  → Respuesta directa
  H3: ¿Pregunta 3?  → Respuesta directa

[CTA final]
  Una sola acción. Texto del botón = verbo + beneficio.
```

## Estructura: Página de Servicio

```
[H1] — Nombre del servicio + para quién o qué resultado produce

[Párrafo intro] — El problema que resuelve. Empatía primero.

[H2] ¿Qué es [servicio] y para qué sirve?
  Definición clara + caso de uso típico.

[H2] ¿Cómo funciona?
  Proceso en 3-5 pasos. Concreto, sin jerga.

[H2] ¿Quién lo necesita?
  Perfil del cliente ideal. Que el lector se reconozca.

[H2] Resultados que puedes esperar
  Beneficios medibles o cambios visibles. Datos si los tienes.

[H2] Preguntas frecuentes sobre [servicio]
  Mínimo 3 FAQs con Schema.

[CTA]
  Acción clara. Sin urgencia falsa.
```

## Estructura: Artículo de Blog / Guía SEO

```
[H1] — Contiene la keyword principal. Responde la intención de búsqueda.
       Formato recomendado: "Cómo [resultado] con [método/herramienta]"
       o "Qué es [tema] y cómo [beneficio]"

[Intro — Párrafo 0]
  3-5 frases. Las 2 primeras responden la pregunta del H1.
  Las siguientes: por qué importa y qué aprenderá el lector.

[Índice — opcional, recomendado en guías largas]

[H2] [Primera subtema — el más importante primero]
  Contenido, ejemplos, datos.

[H2] [Segunda subtema]
  ...

[H2] Preguntas frecuentes sobre [tema del artículo]
  H3: Pregunta → Respuesta directa (2-4 frases)

[Conclusión — breve]
  Resumen en 3-4 líneas. No repite todo el artículo.
  Refuerza el beneficio principal.

[CTA contextual]
  Relacionado con el tema del artículo.
```

## Estructura: Landing Page de Conversión

```
[ABOVE THE FOLD — lo que se ve sin hacer scroll]
  H1: Promesa principal (beneficio claro)
  Subtítulo: Cómo lo conseguimos / para quién
  CTA primario: botón visible sin scroll

[SECCIÓN DE PROBLEMA]
  Describe el dolor del cliente con sus propias palabras.
  Empático, no dramático.

[SECCIÓN DE SOLUCIÓN]
  Cómo lo resuelves. En términos de resultado, no de características.

[SECCIÓN DE CÓMO FUNCIONA]
  3-5 pasos simples. Visual si es posible.

[PRUEBA SOCIAL]
  Testimonios reales, datos, casos de uso, logos.

[OBJECIONES]
  Las 3 dudas más comunes, respondidas directamente.
  Formato FAQ.

[CTA FINAL]
  Repetir el CTA principal. Una sola acción.
  Añadir: garantía, sin compromiso, dato que reduce el riesgo.
```

## Estructura: Sección "Sobre nosotros"

```
[H1] — No empieces con "Somos". Empieza con el "por qué".
       Ejemplo: "Creemos que la tecnología debería trabajar para las personas, no al revés."

[Párrafo de visión / manifiesto]
  3-5 frases. Brand Copy. La historia de por qué existe la empresa.
  Protagonista: el cliente y el problema que quería resolver el fundador.

[H2] Qué hacemos
  Technical Copy. Claro, sin jerga. Orientado a resultado.

[H2] Cómo lo hacemos
  Metodología o valores en acción. Concreto, no abstracto.

[H2] El equipo (si aplica)
  Personas reales. Bio breve con credenciales relevantes.
  Importante para E-E-A-T y confianza.

[CTA de contacto o comunidad]
```

## Longitudes recomendadas por formato

| Formato | Longitud ideal |
|---|---|
| Página principal (home) | 600-1.000 palabras visibles |
| Página de servicio | 800-1.500 palabras |
| Artículo de blog corto | 800-1.200 palabras |
| Guía completa | 1.500-3.000 palabras |
| Landing de conversión | 400-800 palabras (más visual) |
| Sección FAQ | 5-10 preguntas, 50-100 palabras cada respuesta |
| Post de LinkedIn | 150-300 palabras |
| Caption de Instagram | 80-150 palabras |
